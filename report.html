Simulation of a Birth and Assassination Process

Ajai Sharma

Introduction

The process being studied is a variant of a branching process. An example of this is the birth and assassination process. Given various input paramaters, lambda and kappa, the above process is simulated and the number of offspring and number of generations a particular family survives is determined.

Simulation Design

Determination of number of children, their birth times, and their assassination times were performed via sampling of the poisson(lambda*t), uniform(t), and exponential(kappa) distributions respectively. The value 't' here represents the length of lifespan of the parent. The values of lambda and kappa ranged from 0.1 to five and from 0 to 2 respectively; These are specified in detail in the table below. Large values of lambda were unable to be tested due to limitations in computational capacity. Termination parameters of the simulation were set at a maximum of 7 generations and a birth time of all children in a particular generation set to 20. This latter value is rather high, so as to allow a complete iteration of seven generations for the simulation.

Findings

And increase in Lambda corresponded with a larger amount of offspring produced, given a small value for kappa. (This is depicted in the first figure below). However, what is noteworthy, is that as kappa gradually increased, the values for the number of offspring all began to converge quite rapidly to a low quantity.

Higher values of Lambda tended to survive for a longer number of generations. However, it is interesting to note that in one particular simulation, when the maximum allowable birth time was set to a lower value, the family did not survive as long as when this limitation was removed. When the simulation was repreated with a larger bound on the maximum allowable birthtime, it was found that a particular family survived for three longer generations (from four to seven).
